# Управление поливом теплицы

> Версия 2026.07.01 Сборка 4  
> Автор: [Softor](https://github.com/S0ft0r)

Blueprint для Home Assistant, реализующий автоматическое управление системой капельного или зонального полива теплицы с поддержкой расписания, прогноза погоды, нескольких направлений полива и аварийной защиты.

## Логика работы

### Подготовка к поливу

За время, определенное в параметра `Время работы насоса`, до планового запуска:

1. Включается насос
2. Для нормально-открытых клапанов — все клапаны направлений закрываются
3. Система готовится к приему команды запуска

Это обеспечивает набор давления в трубопроводе до начала полива.

---

### Запуск полива

В момент планового времени:

1. Открывается главный клапан подачи воды
2. При необходимости, если задано, выдерживается пауза для полного открытия клапана
3. Последовательно для каждого клапана направления:

```
	→ открыть клапан направления N
    → подождать полного открытия клапана
    → осуществить полив по направлению в течение заданного времени
    → закрыть клапан направления N
    → подождать полного закрытия клапана
    → перейти к направлению N+1
```

4. После обхода всех направлений — закрывается главный клапан
5. При необходимости, если задано, выдерживается пауза для полного закрытия клапана

> **Важно:** система работает последовательно. В один момент времени открыто только одно направление.

---

### Выбор клапанов для полива

| Ситуация | Какие клапаны используются |
|---|---|
| Клапаны, используемые утром, не заданы | все клапаны |
| Клапаны, используемые утром, заданы | только эти клапаны |
| Клапаны, используемые вечером, не заданы | все клапаны |
| Клапаны, используемые вечером, заданы | только эти клапаны |

---

### Типы клапанов направления

Параметр, определяющие тип клапанов:

| Тип | Состояние без питания | Действие для "открыть поток" | Действие для "закрыть поток" |
|---|---|---|---|
| НО (нормально-открытый) | Открыт | `turn_on` / `open_valve` | `turn_off` / `close_valve` |
| НЗ (нормально-закрытый) | Закрыт | `turn_off` / `close_valve` | `turn_on` / `open_valve` |

---

### Режимы расписания

#### Ежедневно
Полив запускается каждый день без исключений.

#### Через день
Полив запускается через день на основе четности текущей даты по Unix timestamp. Четность фиксирована глобально и не привязана к дате первого запуска.

#### Выбор дней
Полив запускается только в выбранные дни недели.

---

### Учет прогноза погоды

Если включен параметр `Учитывать прогноз погоды` и задан `Прогнозный сенсор`:

1. Перед каждым запуском система запрашивает почасовой прогноз
2. Проверяются условия погоды на ближайшие несколько часов (Их количество определяется параметром `Количество часов`)
3. Если хотя бы в одном интервале прогноза присутствует условие из `Прогноза погоды` — полив отменяется

Доступные условия для отмены:

| Значение | Описание |
|---|---|
| `rainy` | Дождь |
| `pouring` | Проливной дождь |
| `lightning-rainy` | Гроза |
| `partlycloudy` | Переменная облачность |
| `cloudy` | Облачно |
| `fog` | Туман |

---

### Ручное управление

#### Ручное включение клапана направления

При ручном включении одного из клапанов направления через интерфейс HA:

1. Если насос выключен — автоматически включается насос с задержкой 
2. Все остальные клапаны направлений закрываются
3. Открывается главный клапан подачи воды
4. Начинается полив через выбранное направление

#### Ручное выключение клапана направления

При ручном выключении одного из клапанов направления:

1. Закрывается главный клапан подачи воды

> **Примечание:** ручные действия определяются по наличию `user_id` в контексте события. Физические кнопки и некоторые интеграции могут не передавать `user_id`.

---

### Останов насоса после полива

После завершения полива насос останавливается только при одновременном выполнении условий:

1. Включен параметр `Учитывать присутствие людей`
2. Ни один человек (объект `person`) не находится дома (в зоне `zone.home` с радиусом ~300 м)
3. Насос в данный момент включен

Если кто-то дома — насос продолжает работать, обеспечивая давление в системе для обычного использования.

---

## Аварийная защита

### Таймер автоотключения

При каждом включении главного клапана автоматически запускается таймер автоотключения

Длительность таймера рассчитывается динамически в зависисмости от количества клапанов направления и задержек их открытия/закрытия.

По истечении таймера:

1. Закрывается главный клапан подачи воды
2. Для НО-клапанов — закрываются все клапаны направлений
3. Если никого нет дома — останавливается насос

### Аварийное выключение насоса

Если насос был выключен внешним образом (защита от перегрева, протечка, ручное отключение) во время работы системы, то запускается штатная последовательность завершения полива

## Сценарии работы

### Сценарий 1: Простой полив без насоса и направлений

```
watering_switch: switch.water_valve
pump_off_auto: timer.watering_timer
```

Система просто включает и выключает один клапан по расписанию.

---

### Сценарий 2: Полив с насосом и несколькими направлениями

```
watering_switch: valve.main_water_valve
waterpump_switch: switch.water_pump
watervalve_switch:
  - switch.zone_1
  - switch.zone_2
  - switch.zone_3
pump_off_auto: timer.watering_safety_timer
```

Система последовательно поливает зоны 1, 2, 3, управляя насосом и главным клапаном.

---

### Сценарий 3: Разные направления утром и вечером

```
watervalve_switch:
  - switch.zone_1
  - switch.zone_2
  - switch.zone_3
morning_watervalve_switch:
  - switch.zone_1
  - switch.zone_2
evening_watervalve_switch:
  - switch.zone_3
```

Утром поливаются зоны 1 и 2, вечером только зона 3.

---

### Сценарий 4: Полив с учетом прогноза погоды

```
set_use_forecast_enable: true
forecast_sensor: weather.home
set_rain_conditions:
  - rainy
  - pouring
  - lightning-rainy
set_rain_number_forecast_hours: 6
```

Полив отменяется если в ближайшие 6 часов ожидается дождь, ливень или гроза.

## Ограничения

- Режим `parallel` позволяет одновременно выполняться нескольким экземплярам автоматизации. При одновременном срабатывании расписания и ручного управления возможны конфликты состояний.
- Режим "через день" рассчитывается от Unix epoch (1 января 1970), а не от даты первого запуска.
- Определение ручного действия основано на `context.user_id`. Физические переключатели и некоторые интеграции могут не передавать этот параметр.
- Клапаны в `morning_watervalve_switch` и `evening_watervalve_switch` рекомендуется выбирать из общего списка `watervalve_switch`, так как логика аварийного отключения и ручного управления работает только с общим списком.
- Прогноз погоды запрашивается только при наличии сенсора типа `weather` и включенном параметре учета прогноза.
