# Управление рулонными шторами

> Версия 2026.05.01 Сборка 5
> Автор: [Softor](https://github.com/S0ft0r)

Blueprint для Home Assistant, реализующий автоматическое управление рулонными шторами по положению солнца с поддержкой защиты от прямых солнечных лучей, учёта прогноза погоды и освещённости, управления через слайдер и ограничений по времени.

## Требования

| Компонент | Описание |
|---|---|
| Home Assistant | 2024.x и выше |
| `cover` | Привод рулонных штор (один или несколько) |
| `sun` | Сенсор солнца (по умолчанию `sun.sun`) |
| `sensor` (illuminance) | Датчик освещённости (опционально) |
| `weather` | Сенсор прогноза погоды (опционально) |
| Aqara Z1 Pro или аналог | Выключатель со слайдером через MQTT или ZHA (опционально) |

### Вручную

1. Скачайте файл blueprint.
2. Поместите его в директорию:
```
config/blueprints/automation/S0ft0r/
```
3. Перейдите в **Настройки → Автоматизации → Blueprints**.
4. Создайте новую автоматизацию на основе blueprint.

## Возможности

- Автоматическое открытие штор на рассвете и закрытие на закате.
- Отсрочка открытия штор до заданного времени (раздельно для будней и выходных).
- Ограничение времени закрытия (не позднее заданного часа).
- Защита от прямых солнечных лучей по азимуту и высоте солнца.
- Учёт прогноза погоды при защите от солнца.
- Учёт датчика освещённости с защитой от частых скачков показаний.
- Управление через внешний слайдер (Aqara Z1 Pro и аналоги, MQTT / ZHA).
- Настраиваемые верхний и нижний пределы открытия/закрытия штор.
- Защита от ошибок срабатывания (повторные команды).
- Поддержка внешних сенсоров (`input_number`) для ключевых параметров.

## Параметры настройки

### Основные параметры

| Тип | Описание |
|---|---|
| `cover` (список) | Приводы штор, которыми нужно управлять |
| `sun` | Датчик солнца (по умолчанию `sun.sun`) |

### Режимы работы

Включаются через мультивыбор в интерфейсе настройки:

| Режим | Описание |
|---|---|
| Защита от прямых лучей солнца | Закрытие/открытие штор по азимуту и высоте солнца |
| Учитывать освещённость | Управление по датчику освещённости |
| Учитывать прогноз погоды | Управление на основе прогноза `weather`-сущности |
| Управление по слайдеру | Управление через Aqara Z1 Pro или аналог |
| Ограничение по времени открытия | Не открывать шторы раньше заданного времени |
| Ограничение по времени закрытия | Закрыть шторы не позднее заданного времени |

### Ограничения по времени

| Параметр | По умолчанию | Описание |
|---|---|---|
| Открыть в будни не ранее | `06:10` | Минимальное время открытия в рабочие дни |
| Открыть в выходные не ранее | `07:10` | Минимальное время открытия в выходные |
| Закрыть не позднее | `21:00` | Максимальное время закрытия на закате |

### Защита от прямых лучей солнца

| Параметр | Тип | Описание |
|---|---|---|
| Шторы для защиты | `cover` (список) | Какие шторы закрывать при защите (если не задано — все) |
| Азимут солнца | число / `input_number` | Угол между направлением на север и солнцем для закрытия |
| Высота солнца | число / `input_number` | Угол возвышения солнца для открытия |

> Азимут и высота солнца могут быть заданы как фиксированное значение или через внешний сенсор `input_number`.

### Прогноз погоды

| Параметр | По умолчанию | Описание |
|---|---|---|
| Прогнозный сенсор | — | Сущность `weather` для определения погоды |
| Условия для защиты | sunny, partlycloudy | При каких условиях активна защита от солнца |

Доступные условия:

| Значение | Описание |
|---|---|
| `sunny` | Ясно |
| `partlycloudy` | Переменная облачность |
| `cloudy` | Облачно |
| `fog` | Туман |

### Освещённость

| Параметр | По умолчанию | Описание |
|---|---|---|
| Датчик освещённости | — | Сенсор `illuminance` |
| Нижний порог | `0` люкс | Ниже этого значения — открыть шторы |
| Верхний порог | `0` люкс | Выше этого значения — закрыть шторы |
| Задержка при скачках | `60` сек | Защита от частых пограничных срабатываний |

### Пределы открытия/закрытия

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| Верхний предел | число / `input_number` | `100%` | Максимальная позиция открытия |
| Нижний предел | число / `input_number` | `0%` | Минимальная позиция закрытия |

### Защита от ошибок

| Параметр | По умолчанию | Описание |
|---|---|---|
| Количество повторов | `2` | Сколько раз повторять команду открытия/закрытия |

> При значении `1` защита от ошибок отключена. При значении `2` и более — каждая команда отправляется повторно с небольшой задержкой для надёжности.

### Управление через слайдер

| Параметр | По умолчанию | Описание |
|---|---|---|
| Выключатель/слайдер | — | Устройство Aqara Z1 Pro или аналог |
| Базовый топик | `zigbee2mqtt/+/action` | MQTT-топик устройства |
| Команда «вверх» | `slider_up` | Payload для движения вверх |
| Команда «вниз» | `slider_down` | Payload для движения вниз |
| Команда «стоп» | `single_wireless` | Payload для остановки |

> Поддерживаются устройства, подключённые через **MQTT** и **ZHA**. Тип подключения определяется автоматически.

## Логика работы

### Открытие и закрытие по солнцу

```
Рассвет (sunrise)
    │
    ├─ Ограничение по времени включено?
    │   ├─ Да: ждём заданного времени (будни / выходные)
    │   └─ Нет: открываем сразу
    │
    ▼
Шторы открыты
    │
    ▼
Закат (sunset) или заданное время закрытия
    │
    ▼
Шторы закрыты
```

### Защита от прямых лучей солнца

```
Солнце взошло
    │
    ├─ Азимут солнца > заданного порога?
    │   ├─ Да: проверяем прогноз и освещённость
    │   │   ├─ Условия совпали → закрываем шторы (для защиты)
    │   │   └─ Условия не совпали → шторы остаются открытыми
    │   └─ Нет: шторы остаются открытыми
    │
    ▼
Высота солнца < заданного порога
    │
    ▼
Открываем шторы (защита больше не нужна)
```

**Как это работает:**

1. Когда азимут солнца превышает заданный угол — солнце начинает светить в окно.
2. Система закрывает шторы (все или только выбранные для защиты).
3. Когда высота солнца опускается ниже заданного угла — прямые лучи больше не попадают в окно.
4. Система открывает шторы обратно.

### Учёт прогноза погоды

Прогноз погоды влияет **только на защиту от прямых лучей солнца**:

- Если текущая погода входит в список «солнечных» условий (по умолчанию: `sunny`, `partlycloudy`) — защита работает.
- Если погода облачная — защита не активируется, шторы остаются открытыми.
- При смене погоды шторы автоматически открываются или закрываются.

### Учёт освещённости

Датчик освещённости работает как **дополнительный фильтр** при защите от солнца:

- Если освещённость выше **верхнего порога** — шторы закрываются.
- Если освещённость ниже **нижнего порога** — шторы открываются.
- Встроена **задержка** для защиты от частых скачков показаний (например, при облачной погоде или с датчиками Xiaomi BLE).

### Управление через слайдер

При подключённом выключателе со слайдером (Aqara Z1 Pro или аналог):

| Действие на устройстве | Результат |
|---|---|
| Провести вверх | Открыть шторы |
| Провести вниз | Закрыть шторы |
| Нажать кнопку | Остановить движение |

Поддерживаются оба способа подключения:
- **MQTT** — через `zigbee2mqtt` (топик определяется автоматически по имени устройства).
- **ZHA** — через `zha_event` (по `device_id`).

### Пределы открытия/закрытия

Позволяют ограничить диапазон движения штор:

- **Верхний предел** — например, `80%` означает, что шторы откроются максимум на 80%.
- **Нижний предел** — например, `10%` означает, что шторы закроются только до 10%.

> Пределы можно задать как фиксированные значения или через `input_number` для динамического изменения.

### Защита от ошибок срабатывания

Команда открытия/закрытия отправляется несколько раз (по умолчанию — 2) с небольшой задержкой между повторами.

При каждом повторе позиция немного корректируется:
- при **открытии**: `верхний_предел - (оставшиеся_повторы)`, потом точная позиция.
- при **закрытии**: `нижний_предел + (оставшиеся_повторы)`, потом точная позиция.

Это помогает корректно завершить движение привода даже при кратковременных сбоях связи.

### Ограничения по времени

#### Ограничение открытия

Если включено — шторы не откроются раньше заданного времени, даже если солнце уже взошло.

Время задаётся раздельно для будней и выходных:
- **Будни** — например, `06:10` (не будить раньше).
- **Выходные** — например, `07:10` (можно поспать).

#### Ограничение закрытия

Если включено — шторы закроются не позднее заданного времени, даже если закат ещё не наступил.

## Сценарии работы

### Сценарий 1: Простое управление по солнцу

```yaml
blinds:
  - cover.living_room_blinds
```

Шторы открываются на рассвете и закрываются на закате.

---

### Сценарий 2: Открытие с отсрочкой

```yaml
blinds:
  - cover.bedroom_blinds
automation_extra:
  - extra_opening_until
set_up_time_workday: "07:00:00"
set_up_time_weekend: "09:00:00"
```

В будни шторы откроются не раньше 07:00, в выходные — не раньше 09:00. Если рассвет позже — откроются по рассвету.

---

### Сценарий 3: Защита от прямых лучей

```yaml
blinds:
  - cover.south_window_1
  - cover.south_window_2
automation_extra:
  - extra_sun_protect
sun_azimuth_config:
  active_choice: угол
  угол: 120
sun_elevation_config:
  active_choice: угол
  угол: 15
```

Когда азимут солнца превышает 120° — шторы на южных окнах закрываются. Когда высота солнца опускается ниже 15° — открываются обратно.

---

### Сценарий 4: Защита от солнца с учётом погоды

```yaml
automation_extra:
  - extra_sun_protect
  - extra_forecast
forecast_sensor: weather.home
forecasts:
  - sunny
  - partlycloudy
```

Защита от солнца работает только при ясной или малооблачной погоде. При облачности шторы остаются открытыми.

---

### Сценарий 5: Защита от солнца с учётом освещённости

```yaml
automation_extra:
  - extra_sun_protect
  - extra_illuminance
illuminance_sensor: sensor.outdoor_illuminance
set_max_illuminance: 40000
set_min_illuminance: 10000
set_illuminance_delay: 120
```

Шторы закрываются при освещённости выше 40000 люкс и открываются при падении ниже 10000 люкс. Задержка 120 секунд защищает от частых срабатываний.

---

### Сценарий 6: Управление через Aqara Z1 Pro (MQTT)

```yaml
automation_extra:
  - extra_slider
slider_device: <device_id выключателя>
basic_topic: "zigbee2mqtt/+/action"
payload_up: "slider_up"
payload_down: "slider_down"
payload_stop: "single_wireless"
```

Слайдер вверх — открыть, вниз — закрыть, кнопка — остановить.

---

### Сценарий 7: Комбинированный

```yaml
blinds:
  - cover.bedroom_blinds
  - cover.kitchen_blinds
blinds_for_protect:
  - cover.kitchen_blinds
automation_extra:
  - extra_sun_protect
  - extra_forecast
  - extra_illuminance
  - extra_opening_until
  - extra_close_until
set_up_time_workday: "06:30:00"
set_up_time_weekend: "08:00:00"
set_down_time: "21:30:00"
forecast_sensor: weather.home
illuminance_sensor: sensor.outdoor_illuminance
set_max_illuminance: 50000
set_min_illuminance: 5000
up_limit_config:
  active_choice: процент
  процент: 90
down_limit_config:
  active_choice: процент
  процент: 5
```

Полная конфигурация:
- Две шторы: спальня и кухня.
- Защита от солнца только для кухонных штор.
- Учёт погоды и освещённости.
- Утром не раньше 06:30 (будни) / 08:00 (выходные).
- Вечером не позднее 21:30.
- Шторы открываются максимум на 90%, закрываются до 5%.

## Ограничения

- Blueprint работает в режиме `queued` (до 10 одновременных вызовов). При очень частых срабатываниях датчика освещённости рекомендуется увеличить параметр задержки.
- Определение будней/выходных основано на `weekday()` — без учёта праздничных дней и интеграции `workday`.
- Управление через слайдер по MQTT привязано к имени устройства в топике `zigbee2mqtt/<имя>/action`. При переименовании устройства в Zigbee2MQTT необходимо пересоздать автоматизацию.
- Азимут и высота солнца проверяются как пороговые значения (больше/меньше), а не как диапазон. Для окон, обращённых в разные стороны, рекомендуется создавать отдельные автоматизации.
- Прогноз погоды влияет только на защиту от прямых лучей солнца, а не на основное открытие/закрытие по рассвету и закату.
- При использовании `input_number` для азимута, высоты солнца или пределов открытия/закрытия — значения читаются в момент срабатывания триггера. Изменение `input_number` не вызывает немедленного пересчёта.
- Повторные команды при защите от ошибок отправляются с задержкой в 1 секунду. Для медленных приводов этого может быть недостаточно.
