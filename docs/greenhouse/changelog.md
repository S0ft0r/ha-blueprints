**Версия 1**

- Добавлена базовая логика автоматического полива теплицы.
- Реализовано управление главным реле / клапаном подачи воды.
- Добавлен запуск полива по времени.
- Добавлен таймер автоматического отключения для защиты от перелива и перегрева оборудования.

---

**Версия 2**

- Добавлена поддержка нескольких клапанов направлений полива.
- Добавлена последовательная обработка направлений.
- Добавлены задержки открытия и закрытия клапанов.
- Добавлена поддержка разных типов исполнительных устройств: `switch` и `valve`.

---

**Версия 3**

- Добавлены отдельные настройки для утреннего и вечернего полива.
- Добавлена настройка длительности полива на каждое направление.
- Добавлены режимы работы:
  - каждый день;
  - через день;
  - выбор дней недели.
- Добавлена возможность использовать отдельные направления только утром или только вечером.
- Добавлена поддержка значений из вспомогательных сущностей `input_datetime` и `input_boolean`.

---

**Версия 4**

- Добавлена поддержка насоса.
- Добавлен предварительный запуск насоса перед началом полива.
- Добавлена задержка для набора давления в системе.
- Добавлена логика отключения насоса после завершения полива.
- Добавлен учет присутствия людей дома при отключении насоса.
- Добавлена обработка ручного включения направления полива.
- Добавлено автоматическое закрытие остальных направлений при ручном выборе одного направления.

---

**Версия 5**

- Добавлен учет прогноза погоды.
- Добавлена отмена полива при выбранных погодных условиях.
- Добавлена настройка глубины прогноза в часах.
- Добавлен автоматический расчет времени защитного таймера с учетом количества направлений.
- Добавлена поддержка нормально-открытых и нормально-закрытых клапанов направлений.
- Добавлены YAML-якоря для повторяющихся действий открытия и закрытия.
- Улучшена структура blueprint: настройки разделены на логические секции.
- Добавлена секция логирования ошибок.
- Добавлена запись ошибок в Logbook.
- Добавлена отправка уведомлений через выбранные `notify`-сущности.
- Начата интеграция диагностических проверок насоса и главного клапана.

---

**Версия 6**

- Добавлена секция учета влажности почвы.
- Добавлена возможность учитывать влажность почвы по отдельным направлениям полива.
- Реализована привязка датчиков влажности к направлениям через два списка:
  - список клапанов направлений с датчиками влажности;
  - список соответствующих датчиков влажности почвы.
- Добавлена поддержка ситуаций, когда датчики влажности есть не у всех направлений.
- Направления без датчика влажности продолжают работать по обычной логике расписания.
- Добавлена возможность пропускать отдельное направление, если влажность почвы выше заданного порога.
- Добавлен глобальный порог влажности для пропуска направления.
- Добавлено логирование пропуска направления из-за высокой влажности почвы.
- Добавлена отправка уведомления при пропуске направления из-за высокой влажности почвы.
- Насос переведен в необязательный параметр с `default: []`.
- Добавлена переменная проверки наличия насоса.
- Добавлены YAML-якоря включения и выключения насоса.
- Улучшена подготовка к работе без отдельного реле насоса.
- Улучшен блок уведомлений: поддержан выбор нескольких `notify`-сущностей.
- Уточнено логирование отмены полива по прогнозу погоды.

---

**Версия 7**

- Добавлен внеплановый полив по датчику влажности почвы при падении ниже заданного порога.
- Добавлено время подтверждения низкой влажности для защиты от ложных срабатываний.
- Добавлена настройка длительности внепланового полива.
- Добавлена асинхронная диагностика датчиков влажности почвы:
  - после каждого полива направления система проверяет прирост влажности;
  - диагностика выполняется в фоне и не увеличивает длительность цикла полива;
  - при отсутствии ожидаемого прироста выполняется логирование и отправка уведомления.
- Добавлена секция адаптации длительности полива по температуре воздуха:
  - ниже холодной границы — полив не запускается;
  - между холодной и комфортной границей — плавное сокращение времени полива;
  - между комфортной и жаркой границей — базовое время полива;
  - выше жаркой границы — увеличенное время полива.
- Добавлен датчик температуры воздуха как источник данных для адаптации.
- Добавлены настройки температурных границ и коэффициентов адаптации.
- Расчет таймера защитного отключения адаптирован с учетом скорректированного времени полива.
- Насос переведен в полностью опциональный режим:
  - якоря `waterpump_on` и `waterpump_off` содержат внутреннюю проверку наличия насоса;
  - все ветки автоматизации корректно работают без реле насоса;
  - проверка состояния насоса перед запуском расписания учитывает его отсутствие.
- Pre-start утром и вечером разделен на две независимые ветки `choose`:
  - ветка с насосом: включение насоса и закрытие клапанов;
  - ветка без насоса: только закрытие клапанов.
- Добавлен якорь `close_no_valve` для повторного использования логики закрытия клапанов направлений.
- Исправлена логика ручного включения направления: насос включается только при `is_waterpump_defined and closed`.
- Исправлена проверка состояния насоса в `check_pump`: используется `is_waterpump_defined`.
- Исправлена ветка внепланового полива: проверка включения насоса защищена флагом `is_waterpump_defined`.
- Добавлена проверка состояния насоса перед запуском утреннего и вечернего расписания.
- Уточнены описания параметров датчика температуры.
