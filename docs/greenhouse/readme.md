# Управление поливом теплицы

> Версия 2026.07.01 Сборка 8
> Автор: [Softor](https://github.com/S0ft0r)

Blueprint для Home Assistant, реализующий автоматическое управление системой капельного или зонального полива теплицы с поддержкой расписания, прогноза погоды, учёта влажности почвы, температурной адаптации, нескольких направлений полива и аварийной защиты.

## Требования

| Компонент | Описание |
|---|---|
| Home Assistant | 2024.x и выше |
| `switch` / `valve` | Реле или клапан включения подачи воды |
| `timer` | Вспомогательный элемент для аварийного автоотключения |
| `switch` / `valve` | Реле насоса (опционально) |
| `switch` / `valve` | Клапаны направлений полива (опционально) |
| `sensor` (moisture) | Датчики влажности почвы (опционально) |
| `sensor` (temperature) | Датчик температуры воздуха (опционально) |
| `weather` | Сенсор погоды для учёта прогноза (опционально) |

> **Рекомендация:** для стабильной работы без интернета все исполнительные
> устройства и датчики должны быть подключены через локальные интеграции
> (ZHA, Zigbee2MQTT, ESPHome, локальный MQTT). Облачные интеграции
> (Tuya, eWeLink и т.п.) не гарантируют работу при отсутствии связи.

## Логика работы

### Подготовка к поливу

За время, определённое параметром `Время работы насоса`, до планового запуска:

1. Если насос задан — включается насос.
2. Для нормально-открытых клапанов — все клапаны направлений закрываются.
3. Система готовится к приёму команды запуска.

Это обеспечивает набор давления в трубопроводе до начала полива.

> Если насос не задан — подготовка выполняется только для клапанов направлений.

---

### Запуск полива

В момент планового времени:

1. Открывается главный клапан подачи воды.
2. Последовательно для каждого клапана направления:

```
→ проверка влажности почвы (если включено)
  → если влажность выше порога — направление пропускается
→ открыть клапан направления N
→ подождать полного открытия клапана
→ осуществить полив по направлению в течение заданного времени
→ закрыть клапан направления N
→ подождать полного закрытия клапана
→ (асинхронно) запустить диагностику датчика влажности
→ перейти к направлению N+1
```

3. После обхода всех направлений — закрывается главный клапан.

> **Важно:** система работает последовательно. В один момент времени
> открыто только одно направление.

---

### Адаптация времени полива по температуре

Если включена функция адаптации и задан датчик температуры, базовое
время полива умножается на коэффициент:

| Температура | Коэффициент | Базовые 10 мин → |
|---|---|---|
| Ниже холодной границы | `0.0` (блокировка) | полив не запускается |
| На холодной границе | `set_cold_multiplier` | например, 5 мин |
| Между холодной и комфортной | плавная интерполяция | от 5 до 10 мин |
| Между комфортной и жаркой | `1.0` | 10 мин |
| На жаркой границе и выше | `set_hot_multiplier` | например, 15 мин |

### Внеплановый полив по влажности почвы

Если включён параметр `Внеплановый полив при низкой влажности` и
влажность почвы на одном из направлений опускается ниже заданного
порога на время, превышающее `Время подтверждения низкой влажности`,
система запускает внеплановый полив соответствующего направления.

#### Условия запуска внепланового полива

Внеплановый полив запускается **только** при одновременном выполнении
всех условий:

1. Учёт влажности почвы включён.
2. Внеплановый полив разрешён.
3. Главный клапан в данный момент закрыт (система не занята плановым поливом).
4. Таймер аварийного автоотключения не активен.
5. Текущее время **не попадает** в ночной период (если ночной запрет включён).

#### Запрет внепланового полива в ночное время

Ночной полив может быть вреден для растений (переохлаждение корневой
зоны, развитие грибковых заболеваний), а также затрудняет оперативное
реагирование на возможные проблемы с оборудованием.

Если включён параметр `Запрет внепланового полива в ночное время`,
система блокирует запуск внепланового полива в заданный ночной период.

| Параметр | По умолчанию | Описание |
|---|---|---|
| Запрет в ночное время | `true` | Включение/выключение ночного запрета |
| Начало ночного периода | `22:00` | После этого времени внеплановый полив запрещён |
| Окончание ночного периода | `06:00` | После этого времени внеплановый полив снова разрешён |

> Ночной период корректно обрабатывает переход через полночь.
> Например, период `22:00 — 06:00` означает запрет с 22:00 текущего
> дня до 06:00 следующего дня.

При блокировке внепланового полива ночью:
- в Logbook записывается сообщение о том, что полив отложен;
- отправляется уведомление (если настроено);
- полив **не откладывается автоматически** до утра — он будет запущен,
  если триггер датчика влажности сработает повторно после окончания
  ночного периода, либо потребность в поливе будет покрыта плановым
  утренним поливом.

> **Примечание:** ночной запрет действует **только** на внеплановый
> полив по датчику влажности. Плановый утренний и вечерний полив
> по расписанию работает независимо от этой настройки.

### Диагностика датчиков влажности

Если включена диагностика, после каждого полива направления:

1. Система отправляет в шину событий данные о направлении, датчике
   и влажности **до** полива.
2. Через заданную задержку (`Задержка проверки после полива`) читается
   текущее значение датчика.
3. Если прирост влажности меньше `Минимального ожидаемого прироста` —
   выполняется логирование и отправка уведомления.

> Диагностика выполняется **асинхронно** и не увеличивает длительность
> цикла полива.

Возможные причины предупреждения:

- неисправность датчика;
- засор линии полива;
- вода не дошла до зоны датчика;
- недостаточная длительность полива.

| Ситуация | Какие клапаны используются |
|---|---|
| Клапаны утром не заданы | все клапаны из общего списка |
| Клапаны утром заданы | только эти клапаны |
| Клапаны вечером не заданы | все клапаны из общего списка |
| Клапаны вечером заданы | только эти клапаны |

---

### Типы клапанов направления

| Тип | Состояние без питания | Действие «открыть поток» | Действие «закрыть поток» |
|---|---|---|---|
| НО (нормально-открытый) | Открыт | `turn_on` / `open_valve` | `turn_off` / `close_valve` |
| НЗ (нормально-закрытый) | Закрыт | `turn_off` / `close_valve` | `turn_on` / `open_valve` |

### Режимы расписания

#### Ежедневно
Полив запускается каждый день без исключений.

#### Через день
Полив запускается через день на основе чётности текущей даты по
Unix timestamp. Чётность фиксирована глобально и не привязана к
дате первого запуска.

#### Выбор дней
Полив запускается только в выбранные дни недели.

---

### Учёт прогноза погоды

Если включён параметр `Учитывать прогноз погоды` и задан
`Прогнозный сенсор`:

1. Перед каждым запуском система запрашивает почасовой прогноз.
2. Проверяются условия погоды на ближайшие N часов
   (параметр `Количество часов`).
3. Если хотя бы в одном интервале прогноза присутствует выбранное
   условие — полив отменяется.

Доступные условия для отмены:

| Значение | Описание |
|---|---|
| `rainy` | Дождь |
| `pouring` | Проливной дождь |
| `lightning-rainy` | Гроза |
| `partlycloudy` | Переменная облачность |
| `cloudy` | Облачно |
| `fog` | Туман |

---

### Защита от устаревших данных

При длительном отсутствии интернета или связи с облачными сервисами
некоторые сенсоры могут перестать обновляться, сохраняя последнее
известное значение.

Система контролирует «свежесть» данных для следующих сенсоров:

| Сенсор | Что происходит при устаревании |
|---|---|
| Прогноз погоды (`weather`) | Прогноз игнорируется, полив продолжается по расписанию |
| Персона (`person`) | Персона считается отсутствующей, насос выключается после цикла |

Время, после которого данные считаются устаревшими, задаётся
параметром **Время «протухания»** в секции «Безопасность»
(по умолчанию — 2 часа).

> При обнаружении устаревшего прогноза погоды система записывает
> предупреждение в Logbook и отправляет уведомление
> (если логирование включено).

### Ручное управление

#### Ручное включение клапана направления

При ручном включении одного из клапанов направления через интерфейс HA:

1. Если насос задан и выключен — автоматически включается насос
   с задержкой.
2. Все остальные клапаны направлений закрываются.
3. Открывается главный клапан подачи воды.
4. Начинается полив через выбранное направление.

#### Ручное выключение клапана направления

При ручном выключении одного из клапанов направления:

1. Закрывается главный клапан подачи воды.

> **Примечание:** ручные действия определяются по наличию `user_id`
> в контексте события. Физические кнопки и некоторые интеграции
> могут не передавать `user_id`.

---

### Останов насоса после полива

После завершения полива насос останавливается только при одновременном
выполнении условий:

1. Насос задан.
2. Включён параметр `Учитывать присутствие людей`.
3. Ни один человек (объект `person`) не находится дома
   (в зоне `zone.home` с радиусом ~300 м).
4. Насос в данный момент включён.

Если кто-то дома — насос продолжает работать, обеспечивая давление
в системе.

> Логика определения присутствия основана на сравнении координат,
> а не на стандартном статусе `home`. Это корректно работает
> в конфигурациях с несколькими Home Assistant и пересекающимися
> локациями персоналий.

> Если данные о местоположении персоны не обновлялись дольше
> заданного времени «протухания», такая персона считается
> отсутствующей. Это позволяет корректно выключать насос
> при потере связи с телефоном или облачным трекером.

---

## Аварийная защита

### Таймер автоотключения

При каждом включении главного клапана автоматически запускается
таймер автоотключения.

Длительность таймера рассчитывается динамически как максимум из:

- суммарного времени полива всех направлений утром
  (с учётом температурной адаптации);
- суммарного времени полива всех направлений вечером
  (с учётом температурной адаптации);
- времени внепланового полива по влажности;
- минимального времени до автоотключения из настроек.

По истечении таймера:

1. Закрывается главный клапан подачи воды.
2. Для НО-клапанов — закрываются все клапаны направлений.
3. Если никого нет дома — останавливается насос.

---

### Аварийное выключение насоса

Если насос был выключен внешним образом (защита от перегрева,
протечка, ручное отключение) во время работы системы —
автоматически запускается штатная последовательность завершения полива.

---

## Сценарии работы

### Сценарий 1: Простой полив без насоса и направлений

```yaml
watering_switch: switch.water_valve
pump_off_auto: timer.watering_timer
```

Система просто включает и выключает один клапан по расписанию.

---

### Сценарий 2: Полив с насосом и несколькими направлениями

```yaml
watering_switch: valve.main_water_valve
waterpump_switch: switch.water_pump
watervalve_switch:
  - switch.zone_1
  - switch.zone_2
  - switch.zone_3
pump_off_auto: timer.watering_safety_timer
```

Система последовательно поливает зоны 1, 2, 3, управляя насосом
и главным клапаном.

---

### Сценарий 3: Разные направления утром и вечером

```yaml
watervalve_switch:
  - switch.zone_1
  - switch.zone_2
  - switch.zone_3
morning_watervalve_switch:
  - switch.zone_1
  - switch.zone_2
evening_watervalve_switch:
  - switch.zone_3
```

Утром поливаются зоны 1 и 2, вечером — только зона 3.

---

### Сценарий 4: Полив с учётом прогноза погоды

```yaml
set_use_forecast_enable: true
forecast_sensor: weather.home
set_rain_conditions:
  - rainy
  - pouring
  - lightning-rainy
set_rain_number_forecast_hours: 6
```

Полив отменяется, если в ближайшие 6 часов ожидается дождь,
ливень или гроза.

---

### Сценарий 5: Полив с учётом влажности почвы

```yaml
set_use_soil_moisture: true
soil_moisture_valves:
  - switch.zone_1
  - switch.zone_2
soil_moisture_sensors:
  - sensor.soil_moisture_zone_1
  - sensor.soil_moisture_zone_2
set_soil_block_schedule_enable: true
set_soil_skip_threshold: 65
set_soil_watering_enable: true
set_soil_watering_low_threshold: 30
set_soil_night_block_enable: true
set_soil_night_start: "22:00:00"
set_soil_night_end: "06:00:00"
```

Направления с влажностью выше 65% пропускаются при плановом поливе.
При падении влажности ниже 30% запускается внеплановый полив, но
только в дневное время (с 06:00 до 22:00). В ночные часы внеплановый
полив блокируется.

---

### Сценарий 6: Полив с температурной адаптацией

```yaml
set_weather_adapt_enable: true
weather_temp_sensor: sensor.outdoor_temperature
set_temp_cold: 5
set_temp_comfort: 18
set_temp_hot: 30
set_cold_multiplier: 0.5
set_hot_multiplier: 1.5
```

При температуре ниже 5°C полив не запускается. При температуре
выше 30°C время полива увеличивается в 1.5 раза.

---

## Ограничения

- Режим `parallel` позволяет одновременно выполняться нескольким
  экземплярам автоматизации. При одновременном срабатывании расписания
  и ручного управления возможны конфликты состояний.
- Режим «Через день» рассчитывается от Unix epoch (1 января 1970),
  а не от даты первого запуска.
- Определение ручного действия основано на `context.user_id`.
  Физические переключатели и некоторые интеграции могут не передавать
  этот параметр.
- Клапаны в `morning_watervalve_switch` и `evening_watervalve_switch`
  рекомендуется выбирать из общего списка `watervalve_switch`, так как
  логика аварийного отключения и ручного управления работает только
  с общим списком.
- Прогноз погоды запрашивается только при наличии сенсора типа
  `weather` и включённом параметре учёта прогноза.
- Привязка датчиков влажности к направлениям выполняется по порядку
  двух списков. Изменение порядка в одном списке без изменения
  в другом нарушит соответствие.
- Диагностика датчиков влажности является информационной и не блокирует
  полив. Параметры `Задержка проверки` и `Минимальный прирост`
  необходимо подбирать опытным путём под конкретный тип грунта
  и оборудования.
- Точность срабатывания расписания — **1 минута**. Секунды в параметрах
  времени игнорируются.
- Внеплановый полив, заблокированный ночным запретом, **не откладывается**
  автоматически до утра. Он будет запущен только если триггер датчика
  влажности сработает повторно после окончания ночного периода.
- При отсутствии интернета облачные устройства (Tuya, eWeLink и т.п.)
  становятся недоступными. Автоматизация корректно работает офлайн
  только при использовании локальных интеграций
  (ZHA, Zigbee2MQTT, ESPHome, локальный MQTT).
- При перезагрузке Home Assistant во время активного полива таймер
  аварийного автоотключения сбрасывается. Рекомендуется использовать
  реле с аппаратным auto-off на уровне прошивки для дополнительной
  защиты от залива.
- Уведомления через облачные сервисы (VK, Push) не доставляются
  при отсутствии интернета. Ошибка отправки не влияет на работу полива
  благодаря `continue_on_error`.
